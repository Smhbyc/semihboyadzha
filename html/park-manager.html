Amusement Park Accountability System

Created On December 2, 2016

This project is created to help to calculate the daily profit of any amusement park. Any manager of an amusement park would easily use this program. The program offers three different language; English, Spanish, and Turkish. It is well-known that amusement parks ask for different amount of price to its varied customers. For example, same entertainment toolsâ€™ price can be different to students that adults. Or some of them even offers extra low prices when annual membership is gained. For this reason, it is difficult to calculate all different profits manually.

After the language options, this program asks for the unique username and password that are given to managers to keep the program away from unintended users. After language is chosen and password and username are entered, the program asks for each specific price discriminated customers and how many tickets are bought by each one of the customer groups for each price gap. For example, for children (under 13 years-old), pricing is almost half price of the adults.

The project is completed using VB.Net and Visual Studio.
